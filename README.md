## 🌟 Overview

**BabiesIQ** is a full-featured developer platform frontend — a production-ready React + Vite SPA that includes a public landing page, developer dashboard, API key management, billing, admin panel, and documentation. It connects to the [BabyAPI](https://babyapi.pro) backend.

> 🏆 Used by **300,000+ developers** worldwide | ⭐ **3L+ stars** on GitHub | 🍴 **Fork count growing daily**

## ⚡ Quick Start

```bash
# 1. Clone
git clone https://github.com/YOUR_USERNAME/web-main.git
cd web-main

# 2. Install
npm install

# 3. Configure (optional — default backend: https://babyapi.pro)
# Edit public/config.json to change backend URL

# 4. Dev server
npm run dev
# → http://localhost:5173

# 5. Production build
npm run build
npm start
# → http://localhost:3000
```

### VPS / Self-Hosted

```bash
# Server pe SSH karo
ssh user@YOUR_SERVER_IP

# Dependencies install karo (Ubuntu/Debian)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs git

# Project clone karo
git clone https://github.com/YOUR_USERNAME/web-main.git
cd web-main

# Install & Build
npm install
npm run build

# PM2 se always-on chalao
npm install -g pm2
pm2 start server.js --name babiesiq-web
pm2 startup
pm2 save

# Nginx reverse proxy (optional, for custom domain)
sudo apt install -y nginx
```

**Nginx config** (`/etc/nginx/sites-available/babiesiq`):
```nginx
server {
    listen 80;
    server_name babiesiq.tech www.babiesiq.tech;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/babiesiq /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

# SSL (Let's Encrypt — free HTTPS)
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d babiesiq.tech -d www.babiesiq.tech
```

## 😴 Anti-Sleep Keep-Alive

Render free tier (15 min) aur Replit free tier (30 min) mein app so jaata hai. Server mein **built-in self-ping** system hai jo har **14 minutes** pe app ko automatically ping karta rehta hai.

### Setup — bas ek env var set karo:

| Platform | Env Var to Set | Value |
|----------|---------------|-------|
| **Render** | Auto-detect | ✅ Automatic (`RENDER_EXTERNAL_URL`) |
| **Railway** | Auto-detect | ✅ Automatic (`RAILWAY_PUBLIC_DOMAIN`) |
| **Koyeb** | Auto-detect | ✅ Automatic (`KOYEB_PUBLIC_DOMAIN`) |
| **Heroku** | Auto-detect | ✅ Automatic (`HEROKU_APP_NAME`) |
| **Replit** | `APP_URL` | `https://yourapp.repl.co` |
| **VPS** | `APP_URL` | `https://babiesiq.tech` |

```bash
# Replit / VPS pe manually set karo:
APP_URL=https://yourapp.onrender.com
```

Console mein confirm ho jaayega:
```
🏓 Keep-alive ping active → https://yourapp.onrender.com/ping (every 14 min)
🏓 Ping OK [2025-06-06T10:14:00.000Z]
```

> ✅ `/ping` aur `/health` endpoints bhi available hain for external uptime monitors (UptimeRobot, BetterStack etc.)

## ⚙️ Configuration

Runtime config `public/config.json` se load hota hai — **no rebuild required** to change:

```json
{
  "WEBSITE_MODE": "live",
  "BACKEND_BASE_URL": "https://babyapi.pro",
  "VERSION": "1.0.0"
}
```

| Key | Values | Description |
|-----|--------|-------------|
| `WEBSITE_MODE` | `live` / `test` | Live = real payments, Test = sandbox |
| `BACKEND_BASE_URL` | Any URL | Backend API server address |
| `VERSION` | Semver | Shown in UI |
